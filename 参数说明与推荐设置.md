# Llama-cpp-vlmforQo 节点参数说明与推荐设置

本文档详细介绍了 ComfyUI-llama-cpp-vlmforQo 插件中各个节点的参数作用和推荐设置，帮助用户更好地使用和调节模型。

## 1. Llama-cpp 模型加载节点

### 功能
加载和初始化 LLM/VLM 模型，是所有其他节点的基础。

### 参数说明

参数名：model 
 作用说明：选择要加载的 LLM 模型文件 
 推荐设置：根据需要选择合适的 GGUF 模型 

参数名：enable_mmproj 
 作用说明：启用后可处理图片输入 
 推荐设置：处理图片时设为 True，纯文本时设为 False 

参数名：mmproj 
 作用说明：对应的视觉编码模型文件 
 推荐设置：启用 mmproj 时选择匹配的模型 

参数名：chat_handler 
 作用说明：选择适合模型的对话格式处理器 
 推荐设置：与模型匹配，如 Qwen3-VL 模型选择 Qwen3-VL 

参数名：n_ctx 
 调节范围：1024-327680 
 作用说明：上下文长度，影响可处理的文本长度 
 推荐设置：低性能设备：4096，高性能设备：8192 

参数名：n_gpu_layers 
 调节范围：-1-1000 
 作用说明：加载到 GPU 的模型层数，-1=全部加载 
 推荐设置：高性能设备：-1，低性能设备：0（CPU模式） 

参数名：vram_limit 
 调节范围：-1-24 
 作用说明：显存限制（GB），-1=无限制 
 推荐设置：通常设为 -1 个人推荐设置为实际显存大小减1（16g就设置成15）

参数名：image_min_tokens 
 调节范围：0-4096 
 作用说明：图片最小编码 token 数 
 推荐设置：保持默认 0 

参数名：image_max_tokens 
 调节范围：0-4096 
 作用说明：图片最大编码 token 数 
 推荐设置：保持默认 0

### 智能推荐
- 系统会根据硬件性能自动推荐 n_ctx 和 n_gpu_layers 的默认值
- 低性能设备会自动降低默认参数以保证运行流畅

## 2. Llama-cpp 图片推理节点

### 功能
进行 LLM/VLM 推理，支持纯文本、单张图片、多张图片和视频输入。

### 参数说明

参数名：llama_model 
 作用说明：加载的 LLM 模型 
 推荐设置：连接到模型加载节点 

参数名：preset_prompt 
 作用说明：预设提示词模板 
 推荐设置：根据需要选择，如描述图片选择 "Normal - Describe" 

参数名：custom_prompt 
 作用说明：自定义提示词 
 推荐设置：与预设提示词配合使用 

参数名：system_prompt 
 作用说明：系统提示词，定义模型行为 
 推荐设置：根据需求修改 

参数名：inference_mode 
 作用说明：推理模式 
 推荐设置：单张图片：images，多张图片：one by one，视频：video 

参数名：max_frames 
 调节范围：2-1024 
 作用说明：视频处理的最大帧数 
 推荐设置：16-32 之间 

参数名：max_size 
 调节范围：128-16384 
 作用说明：图片/视频的最大尺寸 
 推荐设置：低性能设备：256，高性能设备：512 

参数名：seed 
 调节范围：0-0xffffffffffffffff 
 作用说明：随机种子 
 推荐设置：0（随机）或固定值 

参数名：force_offload 
 作用说明：推理后强制卸载模型 
 推荐设置：通常设为 False 

参数名：save_states 
 作用说明：保存会话状态 
 推荐设置：连续对话时设为 True 

参数名：parameters 
 作用说明：连接参数设置节点 
 推荐设置：可选，不连接时使用默认参数 

参数名：images 
 作用说明：输入图片 
 推荐设置：处理图片时提供

### 预设提示词模板说明

预设提示词模板提供了多种常用的推理场景：

- **Empty - Nothing**：空模板，完全使用自定义提示词
- **Normal - Describe**：简单描述图片内容
- **Prompt Style - Tags**：生成图片标签列表，适用于SDXL模型提示词
- **Prompt Style - Simple**：生成简洁的图片描述300字以内
- **Prompt Style - Detailed**：生成详细的图片描述600字以内
- **Prompt Style - Comprehensive Expansion**：详细扩写提示词，增强其在AI生成任务中的清晰度与表现力，确保输出语言与输入语言一致1000字以内
- **Creative - Detailed Analysis**：详细分析图片内容
- **Creative - Summarize Video**：总结视频内容的关键事件和叙事点
- **Creative - Short Story**：基于图片或视频生成短篇故事
- **Creative - Refine & Expand Prompt**：优化并扩写用户提示词，使其更具表现力和视觉丰富性
- **Vision - *Bounding Box**：生成物体检测的边界框

## 3. Llama-cpp 参数设置节点

### 功能
设置 LLM 推理的详细参数，控制生成文本的质量和风格。

### 参数说明

参数名：max_tokens 
 调节范围：0-4096 
 作用说明：最大生成 token 数，控制生成文本的最大长度，直接影响输出内容的完整性 
 推荐设置：低性能设备：512-1024，高性能设备：1024-2048 

参数名：top_k 
 调节范围：0-1000 
 作用说明：采样候选词数量，控制每次生成时考虑的词汇数量，越小越聚焦，越大越多样 
 推荐设置：低性能设备：20，高性能设备：30 

参数名：top_p 
 调节范围：0.0-1.0 
 作用说明：核采样阈值，控制生成内容的多样性和准确性平衡，值越大生成越多样 
 推荐设置：0.85-0.9 

参数名：min_p 
 调节范围：0.0-1.0 
 作用说明：最小采样概率，避免低概率词被完全忽略，增加生成的丰富性，默认值通常已足够 
 推荐设置：0.05 

参数名：typical_p 
 调节范围：0.0-1.0 
 作用说明：典型性采样参数，控制生成内容的典型性，值为1.0时不使用此特性，大多数用户无需调整 
 推荐设置：1.0 

参数名：temperature 
 调节范围：0.0-2.0 
 作用说明：生成温度，控制生成的随机性，是最常用的参数之一，越高越随机，越低越确定 
 推荐设置：0.6-0.8 

参数名：repeat_penalty 
 调节范围：0.0-10.0 
 作用说明：重复惩罚，避免生成重复内容，提高文本质量，是常用参数之一 
 推荐设置：1.0 

参数名：frequency_penalty 
 调节范围：0.0-1.0 
 作用说明：频率惩罚，减少高频词出现的概率，默认值0.0表示不使用此功能，通常不需要主动调整 
 推荐设置：0.0 

参数名：presence_penalty 
 调节范围：0.0-2.0 
 作用说明：出现惩罚，鼓励生成新内容，增加文本丰富度，是常用参数之一 
 推荐设置：1.0 

参数名：mirostat_mode 
 调节范围：0-2 
 作用说明：Mirostat采样模式，用于控制生成文本的困惑度，0=关闭，1=基本模式，2=版本2，大多数用户不需要使用 
 推荐设置：0 

参数名：mirostat_eta 
 调节范围：0.0-1.0 
 作用说明：Mirostat学习率，仅在mirostat_mode启用时有效，控制调整速度 
 推荐设置：0.1 

参数名：mirostat_tau 
 调节范围：0.0-10.0 
 作用说明：Mirostat目标困惑度，仅在mirostat_mode启用时有效，控制生成的多样性 
 推荐设置：5.0 

参数名：state_uid 
 调节范围：-1-999999 
 作用说明：对话状态ID，用于保存和恢复对话会话状态，仅在需要保持连续对话上下文时有用 
 推荐设置：-1

### 参数调节技巧

1. **控制输出长度**：调整 `max_tokens`，值越大输出越长，但占用更多资源。建议根据实际需求设置，避免设置过大导致性能问题

2. **控制随机性**：`temperature` 是最常用的参数之一，越低越确定（适合需要准确答案的场景），越高越有创意（适合生成故事、诗歌等创造性内容）

3. **控制多样性与准确性平衡**：
   - `top_p` 和 `top_k` 通常配合使用
   - 降低 `top_k` 并提高 `top_p` 可以在保证一定多样性的同时提高准确性
   - 提高 `top_k` 并降低 `top_p` 可以增加生成的多样性

4. **避免重复内容**：
   - 增加 `repeat_penalty` 可以减少重复内容
   - 结合 `presence_penalty` 使用可以鼓励模型生成新内容

5. **参数优先级建议**：
   - 优先调整：`temperature`、`max_tokens`、`top_p`/`top_k`
   - 次要调整：`repeat_penalty`、`presence_penalty`
   - 高级用户：`min_p`、`mirostat` 相关参数

6. **常用场景参数设置**：
   - **事实问答**：temperature=0.1-0.3, top_k=10, top_p=0.7
   - **创意写作**：temperature=0.8-1.2, top_k=50, top_p=0.95
   - **代码生成**：temperature=0.2-0.4, top_k=20, top_p=0.8
   - **对话交互**：temperature=0.6-0.8, top_k=30, top_p=0.9

## 4. Llama-cpp 清理会话节点

### 功能
清理指定的对话会话状态，释放资源。

### 参数说明

参数名：any 
 作用说明：占位参数，可连接任意节点 
 推荐设置：任意连接 

参数名：state_uid 
 调节范围：-1-999999 
 作用说明：要清理的对话状态 ID 
 推荐设置：-1（清理当前会话）或指定 ID

### 功能

### 参数说明

参数名：any 
 作用说明：占位参数，可连接任意节点 
 推荐设置：任意连接

## 6. JSON 转 Bounding Box 节点

### 功能
将模型生成的 JSON 格式边界框数据转换为可视化的边界框。

### 参数说明

参数名：json 
 作用说明：包含边界框信息的 JSON 字符串 
 推荐设置：连接推理节点的输出 

参数名：mode 
 作用说明：边界框解析模式 
 推荐设置：根据模型选择，Qwen3-VL 或 Qwen2.5-VL 

参数名：label 
 作用说明：筛选特定标签的边界框 
 推荐设置：留空表示不筛选 

参数名：image 
 作用说明：原始图片 
 推荐设置：可选，用于绘制边界框

## 二、设备性能适配建议

### 低性能设备（如 3060 6G）

- **模型加载节点**：
  - n_ctx: 4096
  - n_gpu_layers: 0（使用 CPU）
  - max_size: 256

- **参数设置节点**：
  - max_tokens: 512-1024
  - temperature: 0.6
  - top_k: 20
  - top_p: 0.85

### 高性能设备（如 4090）

- **模型加载节点**：
  - n_ctx: 8192
  - n_gpu_layers: -1（全部加载到 GPU）
  - max_size: 512-1024

- **参数设置节点**：
  - max_tokens: 1024-2048
  - temperature: 0.8
  - top_k: 30
  - top_p: 0.9

## 三、常见问题与解决方案

### 1. 显存不足（OOM）

**症状**：推理时提示 "out of memory" 或 "OOM"

**解决方案**：
- 降低 `n_gpu_layers` 值，减少 GPU 负载
- 减少 `n_ctx` 值，降低上下文长度
- 降低 `max_tokens` 值，减少生成文本长度
- 使用更小的模型文件
- 降低图片 `max_size` 值

### 2. 推理速度慢

**解决方案**：
- 增加 `n_gpu_layers` 值（如果有足够显存）
- 减少 `n_ctx` 值
- 使用更小的模型
- 关闭不必要的应用程序，释放系统资源

### 3. 生成结果质量差

**症状**：生成的文本质量低、重复、不相关或不符合预期

**解决方案**：

1. **优化提示词**：好的提示词比参数调节更重要，确保提示词清晰、具体，提供足够的上下文信息

2. **调整生成参数**：
   - 如果生成内容过于随机或不相关：降低 `temperature`，减小 `top_p`/`top_k`
   - 如果生成内容过于重复：增加 `repeat_penalty`，提高 `presence_penalty`
   - 如果生成内容不够丰富：提高 `temperature`，增加 `top_p`/`top_k`，考虑调整 `min_p`

3. **增加输出长度**：如果生成内容被截断，增加 `max_tokens` 值

4. **使用更高质量的模型**：尝试使用更大或更专业的模型

5. **检查模型适配性**：确保选择的 `chat_handler` 与使用的模型类型匹配

### 4. 处理图片时出错

**解决方案**：
- 确保已启用 `mmproj` 并选择了正确的视觉编码模型
- 确保 `chat_handler` 与模型匹配
- 检查图片尺寸是否过大，降低 `max_size` 值

## 四、使用建议

1. **逐步调节**：先使用默认参数，然后根据实际效果逐步调整
2. **模型匹配**：确保 `chat_handler` 与使用的模型类型匹配
3. **资源监控**：使用任务管理器监控内存和显存使用情况
4. **提示词优化**：好的提示词比参数调节更重要，建议详细描述需求
5. **会话管理**：长时间使用时定期清理会话状态，释放资源
