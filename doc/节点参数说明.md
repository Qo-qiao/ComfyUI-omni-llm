# ComfyUI-omni-llm 节点参数说明与推荐设置

本文档详细介绍了 ComfyUI-omni-llm 插件中各个节点的参数作用和推荐设置，帮助用户更好地使用和调节模型。

## 动态模型支持

本插件实现了智能模型检测系统，能够自动发现并支持llama_cpp_python新增的VL模型。当llama_cpp_python更新并支持新的模型版本时，用户可直接下载使用，无需修改插件代码。

## 文档结构

本文档按照以下结构组织：
- **节点参数说明**：详细介绍每个节点的参数及其作用
- **设备性能适配建议**：基于不同硬件配置的推荐设置
- **支持的模型说明**：各种模型的特点和适用场景
- **常见问题与解决方案**：使用过程中可能遇到的问题及解决方法
- **使用建议与最佳实践**：提高模型性能和结果质量的技巧

## 1. 模型加载节点

### 1.1 Llama-cpp 模型加载节点 (llama_cpp_model_loader)

### 功能
加载和初始化 LLM（大语言模型）/VLM（视觉语言模型）模型，是所有其他节点的基础。支持.gguf格式，自动检测多分段模型。

### 参数说明

#### 基础参数

参数名：**模型文件** 
- 作用说明：选择要加载的 LLM 模型文件 
- 推荐设置：根据需要选择合适的 GGUF 模型 
- 支持格式：.gguf

参数名：**启用多模态功能 (enable_mmproj)**
- 作用说明：启用后自动选择对话格式处理器并启用多模态功能
- 推荐设置：处理图片或需要对话格式时设为 True，纯文本生成时设为 False
- 注意：启用后需要选择对应的视觉编码模型

参数名：**视觉编码模型 (mmproj)**
- 作用说明：选择对应的视觉编码模型文件
- 推荐设置：启用多模态功能时选择匹配的模型
- 支持格式：.gguf
- 注意事项：
  - 不同模型需要对应的 mmproj 文件，确保版本匹配

参数名：**启用ASR (enable_asr)**
- 作用说明：启用ASR语音识别功能
- 推荐设置：需要语音识别时设为 True
- 前提条件：需配合ASR模型加载器使用

参数名：**启用TTS (enable_tts)**
- 作用说明：启用TTS语音合成功能
- 推荐设置：需要语音合成时设为 True
- 前提条件：需配合TTS模型加载器使用

#### 运行模式参数

参数名：**上下文长度 (n_ctx)**
- 调节范围：1024-327680
- 作用说明：上下文长度，影响可处理的文本长度
- 推荐设置：
  - 24GB+显存（5090/4090）：16384
  - 16GB显存（4080）：8192
  - 12GB显存（4070 Ti/3080）：6144
  - 8GB显存（3070/3060）：4096
  - 4-6GB显存：2048
- 特殊模型要求：
  - Qwen3系列：GPU模式建议≤4096,  Qwen3.5建议≤2048
- 经验法则：上下文长度与任务复杂度成正比，与硬件性能成反比

参数名：**GPU模型层数 (n_gpu_layers)**
- 调节范围：-1-1000
- 作用说明：加载到GPU的模型层数，-1=全部加载（GPU模式有效）
- 推荐设置：
  - GPU 模式：
    - 24GB+显存（5090/4090）：-1（全部加载）
    - 16GB显存（4080）：-1（全部加载）
    - 12GB显存（4070 Ti/3080）：-1（全部加载）
    - 8GB显存（3070/3060）：30（部分加载）

参数名：**显存限制 (vram_limit)**
- 调节范围：-1-24
- 作用说明：显存限制（GB），-1=无限制（GPU模式有效）
- 推荐设置：
  - GPU 模式：通常设为 -1 或实际显存大小减1（16g就设置成15）

#### 图像处理参数

参数名：**图片最小编码token数 (image_min_tokens)**
- 调节范围：0-4096
- 作用说明：图片最小编码token数
- 推荐设置：
  - 默认：0（自动）
  - Qwen3系列：≥1024（自动调整）
  - MiniCPM-o-4.5：≥768（自动调整）
- 注意事项：某些模型需要最小token数才能正常处理图像

参数名：**图片最大编码token数 (image_max_tokens)**
- 调节范围：0-4096
- 作用说明：图片最大编码token数
- 推荐设置：保持默认 0（自动）
- 注意事项：如设置，必须≥image_min_tokens

#### 高级参数

参数名：**注意力类型 (attention_type)**
- 调节范围：Auto/Standard/Flash/XFormers
- 作用说明：选择模型使用的注意力机制类型，影响模型推理速度和内存使用
- 推荐设置：Auto（默认）
- 选项说明：
  - Auto：自动选择（推荐，NVIDIA GPU自动启用Flash Attention）
  - Flash：Flash Attention（NVIDIA GPU专用优化，推理速度最快）
  - XFormers：XFormers优化（AMD、Intel GPU专用优化，内存使用更高效）
- 使用场景：遇到注意力机制兼容性问题或需要优化性能时手动调整

### 智能推荐
- 系统会根据硬件性能自动推荐 device_mode、n_ctx 和 n_gpu_layers 的默认值
- 低性能设备会自动降低默认参数以保证运行流畅
- CPU 模式会自动忽略 GPU 相关参数，无需手动调整
- 批处理参数（n_batch, n_ubatch, n_threads, n_threads_batch）已改为自动根据硬件性能调整


## 2. Llama-cpp 统一推理节点 (llama_cpp_unified_inference)

### 功能
进行 LLM/VLM/Omni 模型推理，支持纯文本、单张图片、多张图片、音频和视频输入，是统一的推理入口。

**注意**：此节点支持所有模型类型（LLM、VLM、Omni），无需区分模型类型。

#### 模型配置

参数名：**Llama模型**
- 作用说明：选择加载的LLM/VLM/Omni模型，用于推理处理
- 推荐设置：连接到模型加载节点
- 支持：所有模型类型（LLM、VLM、Omni）

#### 推理模式

参数名：**推理模式 (inference_mode)**
- 作用说明：选择推理模式，决定节点处理的输入类型和任务类型
- 选项：
  - `[基础] 文本生成 (Text Generation)`：纯文本模式，仅处理文本输入
  - `[基础] 图像理解 (Image Understanding)`：图像处理模式，处理单张或多张图片
  - `[基础] 音频转文本 (Audio to Text)`：使用ASR模型将音频转换为文本
  - `[基础] 文本转音频 (Text to Audio)`：生成文本后使用TTS模型转换为语音
  - `[高级] 视频理解 (Video Understanding)`：处理视频文件，提取帧进行分析


#### 语言设置

参数名：**预设模板语言 (prompt_language)**
- 作用说明：调用对应语言的预设模板
- 选项：中文 / English
- 推荐设置：根据需要选择

参数名：**输出语言 (response_language)**
- 作用说明：选择输出文本的语言
- 选项：中文 / English
- 推荐设置：根据需要选择

参数名：**ASR语言 (asr_language)**
- 作用说明：ASR语音识别目标语言
- 选项：自动检测 / 中文 / English / 日语 / 韩语 / 法语 / 德语 / 西班牙语
- 推荐设置：根据音频内容选择

#### 视频处理参数

参数名：**最大帧数 (video_max_frames)**
- 调节范围：2-1024
- 作用说明：视频处理的最大帧数
- 推荐设置：16-32 之间
- 性能影响：帧数越多，分析越全面，但处理时间越长

参数名：**抽帧模式 (video_sampling)**
- 作用说明：视频帧采样方式
- 选项：
  - `自动均匀采样`：从视频中均匀抽取指定数量的帧
  - `手动指定帧索引`：自定义要提取的帧号
- 推荐设置：自动均匀采样（默认）

参数名：**手动帧索引 (video_manual_indices)**
- 作用说明：手动模式下的帧索引，仅在手动采样时生效
- 推荐设置：根据需要输入帧索引，如 "0,5,10,15" 或 "0-10"
- 格式要求：使用逗号分隔的数字

#### 图像处理参数

参数名：**最大尺寸 (image_max_size)**
- 调节范围：128-16384
- 作用说明：图像处理的最大边长（像素）
- 推荐设置：
  - 低性能设备：256
  - 高性能设备：512
- 平衡建议：尺寸越大，细节捕捉越好，但显存占用越高

#### 生成参数

参数名：**随机种子 (seed)**
- 调节范围：0-0xffffffffffffffff
- 作用说明：随机种子，用于复现结果
- 推荐设置：0（随机）或固定值

参数名：**强制卸载 (force_offload)**
- 作用说明：推理完成后强制卸载模型释放显存
- 推荐设置：通常设为 False
- 使用场景：仅在需要立即释放显存时使用

参数名：**保存状态 (save_states)**
- 作用说明：保存模型状态以便恢复
- 推荐设置：连续对话时设为 True
- 优势：保持对话上下文，提高多轮交互的连贯性

#### 可选输入

参数名：**参数 (parameters)**
- 作用说明：额外的生成参数配置
- 推荐设置：可选，不连接时使用默认参数
- 来源：连接到参数设置节点

参数名：**图像 (images)**
- 作用说明：图像输入（用于图像理解模式）
- 推荐设置：处理图片时提供

参数名：**视频 (video)**
- 作用说明：视频输入（用于视频理解模式）
- 推荐设置：处理视频时提供

参数名：**音频 (audio)**
- 作用说明：音频输入（用于ASR识别）
- 推荐设置：处理音频时提供

参数名：**TTS模型 (tts_model)**
- 作用说明：TTS模型输入（用于语音合成）
- 推荐设置：需要语音输出时连接到TTS加载器

参数名：**ASR模型 (asr_model)**
- 作用说明：ASR模型输入（用于语音识别）
- 推荐设置：需要语音识别时连接到ASR加载器

### 输出说明

- **output**：生成的文本输出
- **output_list**：生成的文本列表（支持批量输出）
- **state_uid**：对话状态ID
- **audio**：生成的音频数据（仅在文本转音频模式下有效）

## 3. Llama-cpp 参数设置节点 (llama_cpp_parameters)

### 功能
设置 LLM 推理的详细参数，控制生成文本的质量和风格。

### 参数说明

参数名：**最大token数 (max_tokens)**
- 调节范围：0-4096
- 作用说明：最大生成token数，影响输出文本长度
- 推荐设置：低性能设备：512-1024，高性能设备：1024-2048
- 实用建议：
  - 简短回答：256-512
  - 详细描述：768-1024
  - 长文本生成：1024-2048

参数名：**采样候选数 (top_k)**
- 调节范围：0-1000
- 作用说明：采样候选数，值越小生成越集中
- 推荐设置：低性能设备：20，高性能设备：30
- 影响分析：
  - 低 top_k（<20）：生成更确定，适合事实性任务
  - 高 top_k（>50）：生成更多样，适合创造性任务

参数名：**核心采样阈值 (top_p)**
- 调节范围：0.0-1.0
- 作用说明：核心采样阈值，控制生成多样性
- 推荐设置：0.85-0.9
- 效果对比：
  - 低 top_p（<0.8）：生成更保守，准确性高
  - 高 top_p（>0.9）：生成更开放，多样性强

参数名：**最小采样概率 (min_p)**
- 调节范围：0.0-1.0
- 作用说明：最小采样概率，避免完全忽略低概率词汇
- 推荐设置：0.05
- 适用场景：需要增加生成多样性时使用

参数名：**典型采样阈值 (typical_p)**
- 调节范围：0.0-1.0
- 作用说明：典型采样阈值，控制生成的典型性
- 推荐设置：1.0

参数名：**生成温度 (temperature)**
- 调节范围：0.0-2.0
- 作用说明：生成温度，值越高越随机
- 推荐设置：0.6-0.8
- 温度指南：
  - 低温度（0.1-0.4）：适合需要准确答案的场景，如问答、代码生成
  - 中温度（0.5-0.8）：适合大多数场景，平衡准确性和创造性
  - 高温度（0.9-1.5）：适合创造性任务，如故事生成、诗歌创作

参数名：**重复惩罚 (repeat_penalty)**
- 调节范围：0.0-10.0
- 作用说明：重复惩罚，避免生成重复内容
- 推荐设置：1.0
- 调整建议：
  - 当出现重复内容时，逐步增加到1.1-1.3
  - 不宜过高，否则可能导致生成内容不连贯

参数名：**频率惩罚 (frequency_penalty)**
- 调节范围：0.0-1.0
- 作用说明：频率惩罚，减少高频词汇出现
- 推荐设置：0.0
- 使用场景：需要减少高频词重复时使用

参数名：**存在惩罚 (presence_penalty)**
- 调节范围：0.0-2.0
- 作用说明：存在惩罚，鼓励生成新内容
- 推荐设置：1.0
- 使用场景：
  - 长文本生成时避免主题漂移
  - 对话中鼓励提出新观点

参数名：**Mirostat模式 (mirostat_mode)**
- 调节范围：0-2
- 作用说明：Mirostat采样模式：0=关闭，1=基础版，2=版本2
- 推荐设置：0
- 高级用户建议：尝试模式1或2以获得更一致的生成质量

参数名：**Mirostat学习率 (mirostat_eta)**
- 调节范围：0.0-1.0
- 作用说明：Mirostat学习率
- 推荐设置：0.1

参数名：**Mirostat目标困惑度 (mirostat_tau)**
- 调节范围：0.0-10.0
- 作用说明：Mirostat目标困惑度
- 推荐设置：5.0

参数名：**对话状态ID (state_uid)**
- 调节范围：-1-999999
- 作用说明：对话状态ID，-1=使用节点唯一ID
- 推荐设置：-1
- 会话管理：
  - 使用不同的state_uid可以同时维护多个独立会话
  - 连续对话时保持相同的state_uid以维持上下文

### 参数调节技巧

1. **控制输出长度**：调整 `max_tokens`，值越大输出越长，但占用更多资源。建议根据实际需求设置，避免设置过大导致性能问题

2. **控制随机性**：`temperature` 是最常用的参数之一，越低越确定（适合需要准确答案的场景），越高越有创意（适合生成故事、诗歌等创造性内容）

3. **控制多样性与准确性平衡**：
   - `top_p` 和 `top_k` 通常配合使用
   - 降低 `top_k` 并提高 `top_p` 可以在保证一定多样性的同时提高准确性
   - 提高 `top_k` 并降低 `top_p` 可以增加生成的多样性

4. **避免重复内容**：
   - 增加 `repeat_penalty` 可以减少重复内容
   - 结合 `presence_penalty` 使用可以鼓励模型生成新内容

5. **参数优先级建议**：
   - 优先调整：`temperature`、`max_tokens`、`top_p`/`top_k`
   - 次要调整：`repeat_penalty`、`presence_penalty`
   - 高级用户：`min_p`、`mirostat` 相关参数

6. **常用场景参数设置**：
   - **事实问答**：temperature=0.1-0.3, top_k=10, top_p=0.7
   - **创意写作**：temperature=0.8-1.2, top_k=50, top_p=0.95
   - **代码生成**：temperature=0.2-0.4, top_k=20, top_p=0.8
   - **对话交互**：temperature=0.6-0.8, top_k=30, top_p=0.9

## 4. API模型节点

### 4.1 API配置管理器 (llama_cpp_api_model_config)

#### 功能
创建和管理API模型配置，支持多种外部API服务商。

#### 节点参数

**必需参数：**
- `model_name`: API模型名称，用于标识此配置
- `api_provider`: API服务商，可选 OpenAI/Ollama/llms-py/vllm-omni/自定义等
- `api_base`: API基础URL
- `api_key`: API密钥（如需要）
- `max_tokens`: 最大生成token数（默认1024）
- `temperature`: 温度参数（默认0.7）

### 4.2 API推理节点 (llama_cpp_api_inference)

#### 功能
使用API进行多媒体推理，支持图像、视频、音频输入，文本、音频输出。

#### 节点参数

**必需参数：**
- `api_config`: API配置（连接到API配置管理器节点）
- `inference_mode`: 推理模式，可选 文本生成/图像理解/音频转文本/文本转音频/全模态整合/视频理解
- `preset_prompt`: 选择预设提示词模板
- `system_prompt`: 系统提示词
- `text_input`: 用户输入文本
- `prompt_language`: 预设提示词的语言，可选 中文/English
- `response_language`: AI回复的语言，可选 中文/English
- `video_max_frames`: 视频模式：最大提取帧数（默认16）
- `video_sampling`: 视频帧采样方式，可选 自动均匀采样/手动指定帧索引
- `video_manual_indices`: 手动模式下的帧索引
- `image_max_size`: 图像处理的最大边长（默认256）
- `tts_voice`: TTS语音合成音色选择
- `tts_emotion`: TTS语音合成情感风格
- `tts_speed`: TTS语速（默认1.0）
- `seed`: 随机种子（默认101）

**可选参数：**
- `images`: 图像输入（用于图像理解模式）
- `video`: 视频输入（用于视频理解模式）
- `audio`: 音频输入（用于音频理解模式）

#### 输出

- `text`: 生成的文本
- `audio`: 生成的音频（当选择文本转音频或全模态整合模式时）

## 5. ASR/TTS 模型参数说明

### 5.1 ASR模型加载节点参数 (llama_cpp_asr_loader)

#### 节点显示选项（可手动调节）

- **ASR模型**：选择ASR模型文件，根据需要选择合适的ASR模型，仅完成适配qwen3-asr
- **GPU模型层数**：加载到GPU的模型层数，24GB+显存：-1，16GB显存：-1，12GB显存：-1，8GB显存：20，调节范围 -1-1000
- **语言**：识别语言，根据音频内容选择，可选 auto/zh/en/ja/ko/fr/de/es
- **任务**：任务类型，transcribe（转录）/translate（翻译成英文），可选 transcribe/translate

#### 音频输入要求

- **支持格式**：WAV、MP3、FLAC等常见音频格式
- **推荐采样率**：16000Hz 或 22050Hz
- **声道**：单声道或立体声（会自动处理）

### 5.2 TTS模型加载节点参数 (llama_cpp_tts_loader)

#### 节点显示选项（可手动调节）

- **TTS模型**：选择TTS模型文件，仅完成适配qwen3-tts
- **GPU模型层数**：加载到GPU的模型层数，24GB+显存：-1，16GB显存：-1，12GB显存：-1，8GB显存：20，调节范围 -1-1000
- **采样率**：音频采样率（Hz），Qwen3-TTS推荐24000Hz，其他模型22050Hz，调节范围 8000-48000
- **音色**：选择音色类型，根据内容和场景选择，根据模型支持的音色选择
- **情绪**：选择情绪风格，根据内容情感选择，可选 默认/开心/悲伤/愤怒/惊讶/平静/兴奋/温柔

#### 可选参数

- **temperature**：生成温度，值越高越随机，默认值 0.7，调节范围 0.1-2.0
- **top_p**：核采样参数，控制生成多样性，默认值 0.9，调节范围 0.1-1.0
- **top_k**：Top-K采样参数，默认值 50，调节范围 1-100
- **repetition_penalty**：重复惩罚，值越高越避免重复，默认值 1.1，调节范围 1.0-2.0
- **max_new_tokens**：最大生成token数，默认值 2048，调节范围 100-4096
- **use_cache**：是否使用KV缓存加速推理，默认值 True，可选 True/False
- **ref_audio_path**：音色克隆参考音路径，默认值 ""（空），WAV格式文件路径
- **pitch**：音高偏移，基于参考音调整，默认值 0.0，调节范围 -5.0-5.0
- **volume**：音量，默认值 1.0，调节范围 0.1-2.0

#### 情绪到emotion的映射

- **默认**：emotion值 default，中性情感
- **开心**：emotion值 happy，语调上扬，积极愉快
- **悲伤**：emotion值 sad，语调低沉，悲伤沉重
- **愤怒**：emotion值 angry，语调强烈，激动愤怒
- **惊讶**：emotion值 surprised，语调起伏，惊讶意外
- **平静**：emotion值 calm，语调平稳，客观冷静
- **兴奋**：emotion值 excited，语调高昂，激动兴奋
- **温柔**：emotion值 gentle，语调柔和，亲切温暖

### 5.3 Qwen3-TTS 模型特殊说明

#### 模型变体

- **VoiceDesign**：检测关键字 voicedesign, voice_design，支持语音设计，通过自然语言指令控制音色
- **CustomVoice**：检测关键字 customvoice, custom_voice，支持音色克隆和标准音色

#### 采样率说明

- **12Hz版本**：12000Hz，路径包含"12hz"自动检测
- **标准版本**：24000Hz，默认使用

#### 音色克隆参考音要求

- **格式**：WAV格式
- **时长**：3~60秒最佳
- **声道**：单声道
- **采样率**：48000Hz
- **质量**：清晰、无噪音、无背景音乐

### 5.4 在推理节点中使用TTS

#### 方式一：TTS音色美化输出模式

1. 在推理节点中选择 `音频输出模式` 为 `TTS音色美化输出`


## 7. 音频时间对齐相关节点

### 7.1 强制对齐模型加载器 (llama_cpp_forced_aligner_loader)

强制对齐模型加载节点用于加载Qwen3-ForcedAligner模型，支持自动检测和下载缺失文件。

#### 节点参数

**必需参数：**
- `forced_aligner_model`: 选择强制对齐模型文件，支持Qwen3-ForcedAligner-0.6B模型
- `n_gpu_layers`: 加载到GPU的模型层数，-1表示全部加载到GPU

**可选参数：**
- `precision`: 模型精度选择，可选 float16/bfloat16/float32，默认 float16
- `batch_size`: 批处理大小，默认32，范围1-128
- `flash_attention`: 是否启用FlashAttention2优化，需要安装flash-attn

#### 支持的模型
- **Qwen3-ForcedAligner-0.6B**: 阿里通义千问强制对齐模型，支持中文语音与文本的精确时间对齐

#### 模型文件要求
- `model.safetensors`: 模型权重文件
- `config.json`: 模型配置文件
- `tokenizer.json`: 分词器配置文件
- `preprocessor_config.json`: 预处理配置文件

#### 自动下载功能
节点会自动检查必备文件，缺失时会根据网络连接情况从ModelScope或Hugging Face自动下载。

### 7.2 强制对齐推理节点 (llama_cpp_forced_aligner_inference)

强制对齐推理节点将音频和文本输入ForcedAligner，输出细粒度时间戳（音素/单词级）。

#### 节点参数

**必需参数：**
- `aligner_model`: 连接强制对齐模型加载器的输出
- `audio`: 输入音频数据
- `text`: 要对齐的文本内容

**可选参数：**
- `sample_rate`: 音频采样率，默认16000Hz，范围8000-48000Hz
- `output_format`: 输出格式，可选 Text/SRT/VTT/JSON，默认Text

#### 输出说明

节点输出三个值：
1. **aligned_text**: 对齐后的文本
2. **timestamps**: 时间戳数据（JSON格式），包含段落、单词和音素级别的时间信息
3. **subtitle_text**: 字幕格式文本（SRT或VTT格式）

#### 时间戳数据结构
```json
{
"task_id": "任务ID",
"audio_info": {
    "audio_duration": 音频时长(秒),
    "audio_sample_rate": 采样率
},
"segments": [
    {
    "start": 开始时间(秒),
    "end": 结束时间(秒),
    "text": "段落文本",
    "words": [
        {
        "word": "单词",
        "start": 开始时间(秒),
        "end": 结束时间(秒),
        "phonemes": [
            {
            "phoneme": "音素",
            "start": 开始时间(秒),
            "end": 结束时间(秒)
            }
        ]
        }
    ]
    }
],
"status": "success/failed",
"error_msg": "错误信息"
}
```

### 7.3 TTS时间对齐节点 (tts_align)

TTS时间对齐节点使用强制对齐推理节点输出的时间戳数据，实现按文本时间轴生成配音，支持精确时间对齐和自然流对齐两种模式。

#### 节点参数

**必需参数：**
- `tts_model`: TTS模型
- `timestamps`: 时间戳数据，可从强制对齐推理节点获取或导入JSON格式文件
- `text`: 要合成的文本内容
- `sample_rate`: 音频采样率，默认24000Hz，范围8000-48000Hz

**可选参数：**
- `speed`: TTS速度，默认1.0，范围0.5-2.0
- `align_mode`: 对齐模式，可选 Exact Timing(精确时间对齐)/Natural Flow(自然流对齐)，默认Natural Flow
- `silence_padding`: 段落后静音填充（秒），默认0.1，范围0.0-1.0

#### 对齐模式说明
- **Exact Timing**: 严格按照时间戳生成音频，确保每个段落的时长与时间戳完全匹配
- **Natural Flow**: 保持自然语音节奏，不强制调整音频时长

#### 输出说明
节点输出两个值：
1. **aligned_audio**: 对齐后的音频
2. **alignment_info**: 对齐信息，包含每个段落的对齐状态和时长信息

### 7.4 角色配置节点 (RoleConfig)

角色配置节点用于配置角色的参数，包括名称、音色、语速、音高、音量等，支持本地TTS模型。

#### 节点参数

**必需参数：**
- `role_name`: 角色名称

**本地模型参数：**
- `voice`: 本地模型音色，可选：
  - Vivian - 明亮略带锐气的年轻女声 (中文)
  - Serena - 温暖柔和的年轻女声 (中文)
  - Uncle_Fu - 音色低沉醇厚的成熟男声 (中文)
  - Dylan - 清晰自然的北京青年男声 (中文北京方言)
- `emotion`: 本地模型情绪，可选 default/happy/sad/angry/surprised/calm/excited/gentle，默认default

**通用参数：**
- `speed`: 语速，默认1.0，范围0.5-2.0
- `pitch`: 音高偏移，默认0.0，范围-5.0-5.0
- `volume`: 音量，默认1.0，范围0.1-2.0

#### 输出说明
节点输出一个值：
- **role_config**: 角色配置字典，包含所有配置参数

## 8. 常见问题与解决方案

### 8.1 TTS音频输出失败

**原因**：
- TTS模型不支持指定的音色或情绪
- 文本长度超过模型限制
- 缺少必要的音频处理依赖

**解决方案**：
- 检查TTS模型支持的音色和情绪范围
- 缩短输入文本长度
- 安装必要的音频处理库

### 8.2 TTS音色克隆效果不佳

**原因**：
- 参考音频质量差
- 参考音频格式不正确
- 参考音频时长不合适

**解决方案**：
- 使用清晰的参考音频，避免噪音和背景音乐
- 确保参考音频为WAV格式，单声道，48000Hz
- 参考音频时长控制在3~60秒

### 8.3 ASR识别准确率低

**原因**：
- 音频质量差
- 语言设置不匹配
- 模型选择不当

**解决方案**：
- 使用清晰的音频文件
- 设置正确的语言参数
- 选择适合语言和场景的ASR模型

### 8.4 音频保存失败

**原因**：
- 音频数据格式不正确
- 输出路径不存在
- 格式不支持

**解决方案**：
- 确保音频数据包含waveform和sample_rate字段
- 检查输出路径是否存在
- 使用支持的格式（wav/mp3/flac/ogg）

## 10. 多图输入节点说明

### 10.1 节点概述

**Multi-Image Input (Story Creation)** 节点支持两种工作模式：

1. **图像模式**：分析多张图片并创作故事内容，支持多种视频生成模型（WAN2.2、LTX2等）

### 10.2 节点功能

#### 主要功能
1. **双模式支持**：图片模式和文本模式灵活切换
2. **多图输入**：支持输入多张图片（图片模式）
3. **自动预处理**：自动缩放和编码图片
4. **故事创作**：生成适合视频生成的故事内容
5. **灵活配置**：支持多种故事类型、长度和风格
6. **多种应用**：支持故事创作、剧本创作、广告文案等多种内容类型
7. **图片输出**：将图片数据传递给推理节点

### 10.3 输入参数

#### 工作模式
- **mode** (下拉选择): 工作模式
  - 图像模式：分析图片内容进行故事创作
  - 文本模式：通过选项设置生成提示词

#### 图片输入（仅图像模式）
- **image1** ~ **image12** (IMAGE): 输入图片1-12张

- **language** (下拉选择): 输出语言（中文 / English）

- **custom_prompt** (文本): 自定义提示词

- **content_focus** (下拉选择): 内容重点
  - 平衡发展/强调情节/强调角色/强调情感/强调视觉/强调对话

- **target_audience** (下拉选择): 目标受众
  - 普通大众/青少年/儿童/专业人士/特定群体

- **video_model** (下拉选择): 视频生成模型类型
  - WAN2.2：强调场景描述和视觉元素
  - LTX2：注重细节描述和情感表达
  - General Video：平衡场景描述和叙事流畅性
  - Custom：自定义视频生成模型

### 10.6 最佳实践

#### 模式选择建议
- **图像模式**：有具体的图片素材需要分析
- **文本模式**：没有图片素材，需要从零创作

#### 图片选择建议
1. **连贯性**：选择内容连贯的图片
2. **数量适中**：建议1-12张图片
3. **质量优先**：使用高质量的图片
4. **场景多样**：包含不同的场景和角度

#### 参数设置建议
1. **内容类型**：
   - WAN2.2：推荐"连贯故事"或"分镜描述"
   - LTX2：推荐"角色发展"或"情感递进"
2. **内容长度**：
   - 短视频（10-30秒）：200字以内
   - 中等视频（30-60秒）：400字以内
   - 长视频（60-120秒）：600字以内

### 10.7 常见问题

**Q1：图像模式和文本模式有什么区别？**
- **图像模式**：需要提供图片，模型分析图片内容并创作故事
- **文本模式**：不需要图片，通过选项设置生成提示词

**Q2：为什么故事不够连贯？**
可能原因：图片内容不够连贯/故事类型选择不当/自定义提示词不够明确
解决方案：选择内容更连贯的图片/尝试不同故事类型/添加更具体的自定义提示词

**Q3：如何正确连接图片数据到推理节点？**
- **图像模式**：prompt→custom_prompt，images→images
- **文本模式**：prompt→custom_prompt，无需连接images
