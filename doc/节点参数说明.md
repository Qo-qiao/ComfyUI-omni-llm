# ComfyUI-omni-llm 节点参数说明与推荐设置

本文档详细介绍了 ComfyUI-omni-llm 插件中各个节点的参数作用和推荐设置，帮助用户更好地使用和调节模型。

## 动态模型支持

本插件实现了智能模型检测系统，能够自动发现并支持llama_cpp_python新增的VL模型。当llama_cpp_python更新并支持新的模型版本时，用户可直接下载使用，无需修改插件代码。

## 文档结构

本文档按照以下结构组织：
- **节点参数说明**：详细介绍每个节点的参数及其作用
- **设备性能适配建议**：基于不同硬件配置的推荐设置
- **支持的模型说明**：各种模型的特点和适用场景
- **常见问题与解决方案**：使用过程中可能遇到的问题及解决方法
- **使用建议与最佳实践**：提高模型性能和结果质量的技巧
- **提示词权重添加技巧**：提示词权重添加的方法和技巧

## 1. 模型加载节点

### 1.1 Llama-cpp 模型加载节点 (llama_cpp_model_loader)

### 功能
加载和初始化 LLM（大语言模型）/VLM（视觉语言模型）模型，是所有其他节点的基础。支持.gguf格式，自动检测多分段模型。

### 参数说明

#### 基础参数

参数名：**模型文件** 
- 作用说明：选择要加载的 LLM 模型文件 
- 推荐设置：根据需要选择合适的 GGUF 模型 
- 支持格式：.gguf

参数名：**启用多模态功能 (enable_mmproj)**
- 作用说明：启用后自动选择对话格式处理器并启用多模态功能
- 推荐设置：处理图片或需要对话格式时设为 True，纯文本生成时设为 False
- 注意：启用后需要选择对应的视觉编码模型

参数名：**视觉编码模型 (mmproj)**
- 作用说明：选择对应的视觉编码模型文件
- 推荐设置：启用多模态功能时选择匹配的模型
- 支持格式：.gguf
- 注意事项：不同模型需要对应的 mmproj 文件，确保版本匹配

参数名：**启用ASR (enable_asr)**
- 作用说明：启用ASR语音识别功能
- 推荐设置：需要语音识别时设为 True

参数名：**启用TTS (enable_tts)**
- 作用说明：启用TTS语音合成功能
- 推荐设置：需要语音合成时设为 True

#### 运行模式参数

参数名：**上下文长度 (n_ctx)**
- 调节范围：1024-327680
- 作用说明：上下文长度，影响可处理的文本长度和预设模板的完整度
- 推荐设置：
  - 24GB+显存（5090/4090）：16384
  - 16GB显存（4080）：8192
  - 12GB显存（4070 Ti/3080）：6144
  - 8GB显存（3070/3060）：4096
  - 4-6GB显存：2048
- **重要提示**：Qwen3系列建议≤4096，Qwen3.5建议≤2048
- **关键影响**：上下文过短会导致预设模板被截断，影响提示词生成质量

参数名：**GPU模型层数 (n_gpu_layers)**
- 调节范围：-1-1000
- 作用说明：加载到GPU的模型层数，-1=全部加载（GPU模式有效）
- 推荐设置：
  - 24GB+显存：-1（全部加载）
  - 16GB显存：-1（全部加载）
  - 12GB显存：-1（全部加载）
  - 8GB显存：30（部分加载）

参数名：**显存限制 (vram_limit)**
- 调节范围：-1-24
- 作用说明：显存限制（GB），-1=无限制（GPU模式有效）
- 推荐设置：通常设为 -1 或实际显存大小减1

#### 图像处理参数

参数名：**图片最大编码token数 (image_max_tokens)**
- 调节范围：0-4096
- 作用说明：图片最大编码token数
- 推荐设置：保持默认 0（自动）
- **自动优化**：MiMo-VL模型会自动设置为256

#### 高级参数

参数名：**注意力类型 (attention_type)**
- 调节范围：Auto/Standard/Flash/XFormers
- 作用说明：选择模型使用的注意力机制类型，影响模型推理速度和内存使用
- 推荐设置：Flash（默认，NVIDIA GPU自动启用Flash Attention）

### 智能推荐
- 系统会根据硬件性能自动推荐 device_mode、n_ctx 和 n_gpu_layers 的默认值
- 低性能设备会自动降低默认参数以保证运行流畅
- CPU 模式会自动忽略 GPU 相关参数，无需手动调整
- 批处理参数（n_batch, n_ubatch, n_threads, n_threads_batch）已改为自动根据硬件性能调整
- **自动优化项**：image_min_tokens、cache_prompt 已改为后台自动判断，无需手动设置


## 2. Llama-cpp 统一推理节点 (llama_cpp_unified_inference)

### 功能
进行 LLM/VLM/Omni 模型推理，支持纯文本、单张图片、多张图片、音频和视频输入，是统一的推理入口。

**注意**：此节点支持所有模型类型（LLM、VLM、Omni），无需区分模型类型。

#### 模型配置

参数名：**Llama模型**
- 作用说明：选择加载的LLM/VLM/Omni模型，用于推理处理
- 推荐设置：连接到模型加载节点

#### 推理模式

参数名：**推理模式 (inference_mode)**
- **影响程度：高**
- 作用说明：选择推理模式，决定节点处理的输入类型和任务类型
- 选项：
  - `[基础] 文本生成 (Text Generation)`：纯文本模式，仅处理文本输入，生成提示词
  - `[基础] 图像理解 (Image Understanding)`：图像处理模式，处理单张或多张图片，反推提示词
  - `[基础] 批量图像理解 (Batch Image Understanding)`：一次性处理多张图片，减少推理调用次数
  - `[基础] 音频转文本 (Audio to Text)`：使用ASR模型将音频转换为文本
  - `[基础] 文本转音频 (Text to Audio)`：生成文本后使用TTS模型转换为语音
  - `[高级] 视频理解 (Video Understanding)`：处理视频文件，提取帧进行分析

#### 提示词配置（核心参数）

参数名：**预设提示词模板 (preset_prompt)**
- **影响程度：极高**
- 作用说明：选择预设提示词模板，决定提示词的生成结构和风格
- **详细说明**：
  - **[Reverse] Tags**：反推XL标签格式提示词，适合图像反推
  - **[Reverse] Describe**：通用图片反推提示词，生成自然描述
  - **[Normal] Expand**：通用提示词文本优化，扩写输入内容
  - **[Anime] Expand Tags**：二次元角色风格文本优化
  - **[Anime] Prompt Expand**：二次元内容文本优化
  - **[Portrait] Asian Female/Male**：真实亚洲人像文本优化
  - **[Portrait] Western Female/Male**：真实欧美人像文本优化
  - **[Design] Art Illustration**：艺术插画文本优化
  - **[Design] Poster Design**：海报设计文本优化
  - **[Design] Scene Design**：场景设计文本优化
  - **[Design] Interior Design**：室内设计文本优化
  - **[Design] Ecommerce Product**：电商产品文本优化
  - **[Edit] Combined**：图像编辑文本优化
  - **[Text to Video] Universal**：扩写视频文本内容
  - **[Design] Ideogram-4**：JSON结构化提示词

参数名：**系统提示词 (system_prompt)**
- **影响程度：高**
- 作用说明：定义AI助手的角色和行为，可包含预设模板占位符
- 默认值："你是一位优秀的AI提示词处理专家。"
- **关键影响**：系统提示词会引导模型的行为模式，影响输出风格

参数名：**用户输入文本 (text_input)**
- **影响程度：高**
- 作用说明：用户输入文本，作为对话的用户消息内容
- **优化建议**：
  - 图像反推：提供详细的视觉元素描述（人物、环境、光线、构图）
  - 提示词扩写：提供核心创意和场景描述，包含情感基调
  - 多领域设计：明确设计类型（海报、UI、人像等关键词）

#### 语言设置

参数名：**预设模板语言 (prompt_language)**
- 作用说明：调用对应语言的预设模板
- 选项：中文 / English

参数名：**输出语言 (response_language)**
- 作用说明：选择输出文本的语言
- 选项：中文 / English

参数名：**ASR语言 (asr_language)**
- 作用说明：ASR语音识别目标语言
- 选项：自动检测 / 中文 / English / 日语 / 韩语 / 法语 / 德语 / 西班牙语

#### 输出格式设置

参数名：**输出格式 (output_format)**
- **影响程度：中**
- 作用说明：控制输出文本的格式
- 选项：
  - `natural`：以自然段落格式输出纯文本内容
  - `structured`：输出JSON结构化文本内容

#### 视频处理参数

参数名：**最大帧数 (video_max_frames)**
- 调节范围：2-1024
- 作用说明：视频处理的最大帧数
- 推荐设置：16-32 之间

参数名：**抽帧模式 (video_sampling)**
- 作用说明：视频帧采样方式
- 选项：自动均匀采样 / 手动指定帧索引

参数名：**手动帧索引 (video_manual_indices)**
- 作用说明：手动模式下的帧索引，仅在手动采样时生效

#### 图像处理参数

参数名：**最大尺寸 (image_max_size)**
- 调节范围：128-16384
- 作用说明：图像处理的最大边长（像素）
- 推荐设置：高性能设备512，低性能设备256

#### 批量输出选项

参数名：**批量模式 (batch_combination)**
- 作用说明：批量模式的结果处理方式
- 选项：separate（每张图片单独输出）/ combined（所有结果合并）

#### 生成参数

参数名：**随机种子 (seed)**
- 作用说明：随机种子，用于复现结果

参数名：**强制卸载 (force_offload)**
- 作用说明：推理完成后强制卸载模型释放显存
- 推荐设置：通常设为 False

参数名：**保存状态 (save_states)**
- 作用说明：保存模型状态以便恢复
- 推荐设置：连续对话时设为 True

#### 可选输入

参数名：**参数 (parameters)**
- 作用说明：额外的生成参数配置，连接到参数设置节点

参数名：**图像 (images)**
- 作用说明：图像输入（用于图像理解模式）

参数名：**视频 (video)**
- 作用说明：视频输入（用于视频理解模式）

参数名：**音频 (audio)**
- 作用说明：音频输入（用于ASR识别）

参数名：**TTS模型 (tts_model)**
- 作用说明：TTS模型输入（用于语音合成）

参数名：**ASR模型 (asr_model)**
- 作用说明：ASR模型输入（用于语音识别）

### 输出说明
- **output**：生成的文本输出
- **output_list**：生成的文本列表（支持批量输出）
- **state_uid**：对话状态ID
- **audio**：生成的音频数据（仅在文本转音频模式下有效）

## 输出格式选项：自然段落与结构化输出

本插件支持两种输出格式类型的提示词模板，为不同使用场景提供灵活性。

### 2.1 自然段落输出 (Natural Paragraph)

自然段落输出将所有提示词元素组织成连贯流畅的段落，不使用显式的字段标记。

**特点：**
- 所有视觉元素（主体，光线、构图、色彩、氛围等）被整合成无缝衔接的段落
- 没有【】或**等字段标签
- 适合直接用于图像/视频生成的提示词
- 一目了然，易于阅读理解
- 适合喜欢叙事风格描述的用户

**示例：**
```
一位美丽的动漫少女，黑色长发，温柔微笑，站在樱花花园中。杰作品质，赛璐珞着色，干净线稿。柔和光线营造梦幻氛围。她穿着白色水手服上衣配红色点缀，动态姿势，头发随风飘动。背景是粉色渐变天空下的飘落樱花花瓣。
```

### 2.2 结构化输出 (Structured Output)

结构化输出使用【】括号来组织信息，将内容分成不同的类别，更容易定位特定细节。

**特点：**
- 每个字段用【】括号清晰标注
- 字段包括：【主体描述】【风格特征】【光线效果】【构图方式】【色彩搭配】【细节刻画】【技术参数】【服装造型】等
- 适合系统性分析和审查
- 更容易修改特定元素而无需重写整个提示词
- 适合需要对提示词组件进行精确控制的用户

**示例：**
```
【主体描述】美丽的动漫少女，黑色长发，温柔微笑
【风格特征】杰作，最佳质量，超精细，动漫风格，赛璐珞着色，干净线稿
【光线效果】柔和光线，温柔日光，轮廓光，梦幻氛围
【构图方式】动态姿势，头发飘动，全身构图，看向观众
【色彩搭配】白色，红色，粉色渐变，柔和粉彩色调
【服装造型】白色水手服上衣，红色点缀，百褶裙，荷叶边
【背景场景】樱花花园，飘落花瓣，粉色渐变天空
```

### 2.4 输出格式选择的工作原理

选择预设模板时：

1. **系统提示词加载**：系统加载 `input_template_natural`（自然模板）和 `input_template_structured`（结构化模板）两种变体
2. **用户输入处理**：分析您的输入内容中的关键词和上下文
3. **模板选择**：根据模板类型：
   - **标准模板**：输出格式由您的选择决定（自然/结构化）
   - **智能模板**（ERNIE、QWEN）：检测关键词以确定输出侧重点
4. **输出生成**：模型按照选定的格式规则生成内容

### 2.5 如何选择正确的输出格式

**选择自然段落当：**
- 您需要直接复制粘贴即可用于图像生成的提示词
- 您喜欢阅读流畅的叙事描述
- 您是提示词工程的新手，希望获得简单直接的输出

**选择结构化输出当：**
- 您需要对特定提示词组件进行精确控制
- 您想轻松修改单个元素（光线、构图等）
- 您正在进行系统性分析或创建提示词变体

**使用关键词检测（智能模板）当：**
- 您正在处理混合内容类型
- 您希望模型智能适应输出侧重点
- 您需要领域特定的优化（海报、漫画、UI、产品等）

## 3. Llama-cpp 参数设置节点 (llama_cpp_parameters)

### 功能
设置 LLM 推理的详细参数，控制生成文本的质量和风格。

### 参数说明

#### 核心参数（强烈建议了解）

参数名：**最大token数 (max_tokens)**
- **影响程度：极高**
- 调节范围：0-4096
- 作用说明：最大生成token数，直接影响输出文本长度
- 推荐设置：
  - 简短回答：256-512
  - 详细描述：768-1024
  - 长文本生成：1024-2048
- **关键影响**：设置过小会导致提示词被截断，设置过大会增加推理时间

参数名：**生成温度 (temperature)**
- **影响程度：极高**
- 调节范围：0.0-2.0
- 作用说明：生成温度，值越高越随机，越低越确定
- 推荐设置：0.6-0.8
- **详细指南**：
  - 低温度（0.1-0.4）：适合需要准确答案的场景，如问答、代码生成
  - 中温度（0.5-0.8）：适合大多数场景，平衡准确性和创造性
  - 高温度（0.9-1.5）：适合创造性任务，如故事生成、诗歌创作

参数名：**核心采样阈值 (top_p)**
- **影响程度：高**
- 调节范围：0.0-1.0
- 作用说明：核心采样阈值，控制生成多样性
- 推荐设置：0.85-0.9
- **效果对比**：
  - 低 top_p（<0.8）：生成更保守，准确性高
  - 高 top_p（>0.9）：生成更开放，多样性强

参数名：**采样候选数 (top_k)**
- **影响程度：高**
- 调节范围：0-1000
- 作用说明：采样候选数，值越小生成越集中
- 推荐设置：高性能设备30，低性能设备20
- **影响分析**：
  - 低 top_k（<20）：生成更确定，适合事实性任务
  - 高 top_k（>50）：生成更多样，适合创造性任务

#### 次要参数（按需调整）

参数名：**重复惩罚 (repeat_penalty)**
- 调节范围：0.0-10.0
- 作用说明：重复惩罚，避免生成重复内容
- 推荐设置：1.0
- 使用场景：当出现重复内容时，逐步增加到1.1-1.3

参数名：**存在惩罚 (presence_penalty)**
- 调节范围：0.0-2.0
- 作用说明：存在惩罚，鼓励生成新内容
- 推荐设置：1.0
- 使用场景：长文本生成时避免主题漂移

参数名：**最小采样概率 (min_p)**
- 调节范围：0.0-1.0
- 作用说明：最小采样概率，避免完全忽略低概率词汇
- 推荐设置：0.05

参数名：**频率惩罚 (frequency_penalty)**
- 调节范围：0.0-1.0
- 作用说明：频率惩罚，减少高频词汇出现
- 推荐设置：0.0

参数名：**典型采样阈值 (typical_p)**
- 调节范围：0.0-1.0
- 作用说明：典型采样阈值，控制生成的典型性
- 推荐设置：1.0

#### 高级参数（高级用户使用）

参数名：**Mirostat模式 (mirostat_mode)**
- 调节范围：0-2
- 作用说明：Mirostat采样模式：0=关闭，1=基础版，2=版本2
- 推荐设置：0（关闭）
- 使用场景：追求更一致的生成质量时尝试模式1或2

参数名：**Mirostat学习率 (mirostat_eta)**
- 调节范围：0.0-1.0
- 作用说明：Mirostat学习率
- 推荐设置：0.1

参数名：**Mirostat目标困惑度 (mirostat_tau)**
- 调节范围：0.0-10.0
- 作用说明：Mirostat目标困惑度
- 推荐设置：5.0

#### 会话管理参数

参数名：**对话状态ID (state_uid)**
- 调节范围：-1-999999
- 作用说明：对话状态ID，-1=使用节点唯一ID
- 推荐设置：-1
- 会话管理：使用不同的state_uid可以同时维护多个独立会话

参数名：**推理预算 (reasoning_budget)**
- 调节范围：-1-1024
- 作用说明：推理预算（用于Qwen3.5-Thinking等支持思考模式的模型）
- 推荐设置：-1（无限制）
- 使用场景：0=关闭思考模式，N=限制N个思考token

### 参数调节技巧

1. **控制输出长度**：调整 `max_tokens`，值越大输出越长，但占用更多资源
2. **控制随机性**：`temperature` 是最常用的参数，越低越确定，越高越有创意
3. **控制多样性与准确性平衡**：`top_p` 和 `top_k` 通常配合使用
4. **避免重复内容**：增加 `repeat_penalty` 可以减少重复内容
5. **参数优先级建议**：
   - 优先调整：`temperature`、`max_tokens`、`top_p`/`top_k`
   - 次要调整：`repeat_penalty`、`presence_penalty`
   - 高级用户：`min_p`、`mirostat` 相关参数
6. **常用场景参数设置**：
   - **事实问答**：temperature=0.1-0.3, top_k=10, top_p=0.7
   - **创意写作**：temperature=0.8-1.2, top_k=50, top_p=0.95
   - **代码生成**：temperature=0.2-0.4, top_k=20, top_p=0.8
   - **对话交互**：temperature=0.6-0.8, top_k=30, top_p=0.9

### 5.2 TTS模型加载节点参数 (llama_cpp_tts_loader)

#### 节点显示选项（可手动调节）

- **TTS模型**：选择TTS模型文件，仅完成适配qwen3-tts
- **GPU模型层数**：加载到GPU的模型层数，24GB+显存：-1，16GB显存：-1，12GB显存：-1，8GB显存：20，调节范围 -1-1000
- **采样率**：音频采样率（Hz），Qwen3-TTS推荐24000Hz，其他模型22050Hz，调节范围 8000-48000
- **音色**：选择音色类型，根据内容和场景选择，根据模型支持的音色选择
- **情绪**：选择情绪风格，根据内容情感选择，可选 默认/开心/悲伤/愤怒/惊讶/平静/兴奋/温柔

#### 可选参数

- **temperature**：生成温度，值越高越随机，默认值 0.7，调节范围 0.1-2.0
- **top_p**：核采样参数，控制生成多样性，默认值 0.9，调节范围 0.1-1.0
- **top_k**：Top-K采样参数，默认值 50，调节范围 1-100
- **repetition_penalty**：重复惩罚，值越高越避免重复，默认值 1.1，调节范围 1.0-2.0
- **max_new_tokens**：最大生成token数，默认值 2048，调节范围 100-4096
- **use_cache**：是否使用KV缓存加速推理，默认值 True，可选 True/False
- **ref_audio_path**：音色克隆参考音路径，默认值 ""（空），WAV格式文件路径
- **pitch**：音高偏移，基于参考音调整，默认值 0.0，调节范围 -5.0-5.0
- **volume**：音量，默认值 1.0，调节范围 0.1-2.0

#### 情绪到emotion的映射

- **默认**：emotion值 default，中性情感
- **开心**：emotion值 happy，语调上扬，积极愉快
- **悲伤**：emotion值 sad，语调低沉，悲伤沉重
- **愤怒**：emotion值 angry，语调强烈，激动愤怒
- **惊讶**：emotion值 surprised，语调起伏，惊讶意外
- **平静**：emotion值 calm，语调平稳，客观冷静
- **兴奋**：emotion值 excited，语调高昂，激动兴奋
- **温柔**：emotion值 gentle，语调柔和，亲切温暖

#### 音色克隆参考音要求

- **声道**：单声道

### 5.4 在推理节点中使用TTS

#### 方式一：TTS音色美化输出模式

1. 在推理节点中选择 `音频输出模式` 为 `TTS音色美化输出`
2. 将TTS模型连接到 `tts_model` 输入端口
3. 选择合适的 `tts_voice`、`tts_emotion` 和 `tts_speed`

1. 使用TTS模型加载器加载TTS模型
2. 调用TTS模型的 `synthesize` 方法：
   ```python
   audio_output = tts_model.synthesize(
       text="要合成的文本",
       speaker_id=0,  # 音色ID
       speed=1.0,     # 语速
       emotion="default"  # 情感风格
   )
   ```
3. 使用音频保存节点保存音频

### 7.2 强制对齐推理节点 (llama_cpp_forced_aligner_inference)

- `aligner_model`: 连接强制对齐模型加载器的输出

**可选参数：**
- `sample_rate`: 音频采样率，默认16000Hz，范围8000-48000Hz
- `output_format`: 输出格式，可选 Text/SRT/VTT/JSON，默认Text

#### 输出说明

节点输出三个值：
1. **aligned_text**: 对齐后的文本
2. **timestamps**: 时间戳数据（JSON格式），包含段落、单词和音素级别的时间信息
3. **subtitle_text**: 字幕格式文本（SRT或VTT格式）

    "audio_duration": 音频时长(秒),

    {
    "start": 开始时间(秒),
    "end": 结束时间(秒),
    "text": "段落文本",
    "words": [

### 7.3 TTS时间对齐节点 (tts_align)

TTS时间对齐节点使用强制对齐推理节点输出的时间戳数据，实现按文本时间轴生成配音，支持精确时间对齐和自然流对齐两种模式。

#### 节点参数

**必需参数：**
- `tts_model`: TTS模型
- `timestamps`: 时间戳数据，可从强制对齐推理节点获取或导入JSON格式文件
- `text`: 要合成的文本内容
- `sample_rate`: 音频采样率，默认24000Hz，范围8000-48000Hz

**可选参数：**
- `speed`: TTS速度，默认1.0，范围0.5-2.0
- `align_mode`: 对齐模式，可选 Exact Timing(精确时间对齐)/Natural Flow(自然流对齐)，默认Natural Flow
- `silence_padding`: 段落后静音填充（秒），默认0.1，范围0.0-1.0

#### 对齐模式说明
- **Exact Timing**: 严格按照时间戳生成音频，确保每个段落的时长与时间戳完全匹配
- **Natural Flow**: 保持自然语音节奏，不强制调整音频时长

#### 输出说明
节点输出两个值：
1. **aligned_audio**: 对齐后的音频
2. **alignment_info**: 对齐信息，包含每个段落的对齐状态和时长信息

### 7.4 角色配置节点 (RoleConfig)

角色配置节点用于配置角色的参数，包括名称、音色、语速、音高、音量等，支持本地TTS模型。

#### 节点参数

**必需参数：**
- `role_name`: 角色名称

**本地模型参数：**
- `voice`: 本地模型音色，可选：
  - Vivian - 明亮略带锐气的年轻女声 (中文)
  - Serena - 温暖柔和的年轻女声 (中文)
  - Uncle_Fu - 音色低沉醇厚的成熟男声 (中文)
  - Dylan - 清晰自然的北京青年男声 (中文北京方言)
- `emotion`: 本地模型情绪，可选 default/happy/sad/angry/surprised/calm/excited/gentle，默认default

**通用参数：**
- `speed`: 语速，默认1.0，范围0.5-2.0
- `pitch`: 音高偏移，默认0.0，范围-5.0-5.0
- `volume`: 音量，默认1.0，范围0.1-2.0

#### 输出说明
节点输出一个值：
- **role_config**: 角色配置字典，包含所有配置参数

## 8. 常见问题与解决方案

### 8.1 TTS音频输出失败

**原因**：
- TTS模型不支持指定的音色或情绪
- 文本长度超过模型限制
- 缺少必要的音频处理依赖

**解决方案**：
- 检查TTS模型支持的音色和情绪范围
- 缩短输入文本长度
- 安装必要的音频处理库

### 8.2 TTS音色克隆效果不佳

**原因**：
- 参考音频质量差
- 参考音频格式不正确
- 参考音频时长不合适

**解决方案**：
- 使用清晰的参考音频，避免噪音和背景音乐
- 确保参考音频为WAV格式，单声道，48000Hz
- 参考音频时长控制在3~60秒

### 8.3 ASR识别准确率低

**原因**：
- 音频质量差
- 语言设置不匹配
- 模型选择不当

**解决方案**：
- 使用清晰的音频文件
- 设置正确的语言参数
- 选择适合语言和场景的ASR模型

### 8.4 音频保存失败

**原因**：
- 音频数据格式不正确
- 输出路径不存在
- 格式不支持

**解决方案**：
- 确保音频数据包含waveform和sample_rate字段
- 检查输出路径是否存在
- 使用支持的格式（wav/mp3/flac/ogg）

## 9. 使用流程示例

### 9.1 TTS语音合成

1. **加载模型**：
   - 使用 `llama_cpp_model_loader` 节点加载LLM模型
   - 使用 `llama_cpp_tts_loader` 节点加载TTS模型（如Qwen3-TTS）

2. **配置统一推理节点**：
   - 使用 `llama_cpp_unified_inference` 节点
   - 选择推理模式：`[基础] 文本转音频 (Text to Audio)`
   - 连接LLM模型到 `Llama模型` 输入
   - 连接TTS模型到 `tts_model` 输入
   - 设置TTS音色（tts_voice）、情感（tts_emotion）和语速（tts_speed）

3. **提供文本输入**：
   - 在 `文本输入` 中输入要合成的文本
   - 选择合适的预设模板

4. **保存音频**：
   - 使用 `llama_cpp_audio_saver` 音频保存节点
   - 连接推理节点的 `audio` 输出到音频保存节点
   - 选择保存格式（推荐WAV）

### 9.2 ASR语音识别

1. **加载模型**：
   - 使用 `llama_cpp_model_loader` 节点加载LLM模型
   - 使用 `llama_cpp_asr_loader` 节点加载ASR模型（如Qwen3-ASR）

2. **配置统一推理节点**：
   - 使用 `llama_cpp_unified_inference` 节点
   - 选择推理模式：`[基础] 音频转文本 (Audio to Text)`
   - 连接LLM模型到 `Llama模型` 输入
   - 连接ASR模型到 `asr_model` 输入
   - 设置ASR语言（asr_language）

3. **提供音频输入**：
   - 连接音频输入到 `audio` 端口

4. **运行识别**：
   - 获取识别文本输出

### 9.3 多人会话合成

1. **加载TTS模型**：
   - 使用 `llama_cpp_tts_loader` 节点加载TTS模型（如Qwen3-TTS）

2. **配置多人对话TTS节点**：
   - 使用 `multi_speaker_tts` 节点
   - 连接TTS模型到 `tts_model` 输入
   - 在 `dialogue_text` 中输入多人对话文本
   - 设置 `output_name` 输出文件名

3. **设置对话参数**：
   - 设置 `default_speaker` 默认说话人
   - 设置 `default_emotion` 默认情绪
   - 调整 `speed` 语速参数
   - 设置 `pause_duration` 说话人之间的停顿时间

4. **生成对话音频**：
   - 运行工作流
   - 音频文件将保存在ComfyUI输出目录的audio子目录

## 11. 提示词权重添加技巧与方式

### 11.1 语义优先级排序

本插件的预设模板采用**语义优先级排序**机制，在生成提示词时会自动按照重要性对不同元素进行排序和强调。

#### 核心优先级规则

以 `THICKPAINT_ROLE_ZH`（次世代CG厚涂3D角色人像）模板为例，其语义权重优先级为：

| 优先级 | 元素类别 | 说明 |
|--------|----------|------|
| 1（最高） | 角色结构与材质质感 | 人体比例、骨骼结构、PBR材质、次表面散射等 |
| 2 | 姿态表情与造型设计 | 动态姿势、情绪表达、服装搭配、饰品设计 |
| 3 | 光影色彩体积 | 主光方向、边缘光、环境光、色彩搭配 |
| 4 | 场景环境 | 室内/室外、空间结构、背景元素 |
| 5（最低） | 渲染/笔刷参数 | 技术参数、风格标签等 |

#### 实际效果

当您输入"古风仙侠少女，蓝色汉服"时，模板会：
1. **优先保证**：角色五官精致度、汉服材质质感、皮肤通透感
2. **其次表现**：少女的姿态（如双手合十祈祷）、表情（温柔虔诚）、发型设计
3. **再次处理**：光影氛围（清晨柔和光线、金色轮廓光）、色彩搭配（浅蓝+金色+白色）
4. **最后补充**：场景环境（古风庭院、樱花飘落）、风格标签（唯美、神秘、艺术手绘）

### 11.2 通过 text_input 强调元素

> **注意**：以下技巧用于**影响 LLM 的提示词生成行为**，让模型在生成时更加关注某些元素。它们并非直接设置图像生成模型（SD/Flux等）的权重语法，而是通过引导 LLM 输出更详细的描述来间接达到强调效果。效果因模型而异，建议通过实验找到最适合的方式。

您可以在 `用户输入文本 (text_input)` 字段中使用以下技巧来强调或弱化特定元素：

#### 方法一：重复关键词

通过重复关键词来增加其重要性：

```
古风仙侠少女，古风仙侠少女，蓝色汉服，蓝色汉服，精致面容
```

#### 方法二：使用强调词汇

在关键词前添加强调词汇：

```
【重点】古风仙侠少女，【必须】蓝色汉服，【强烈要求】精致面容，【弱化】背景环境
```

#### 方法三：使用形容词叠加

通过多个形容词描述同一元素来增强其权重：

```
一位极其美丽、面容精致、五官端正、气质高雅的古风仙侠少女
```

#### 方法四：明确指令

直接告诉模型哪些元素更重要：

```
古风仙侠少女，重点突出人物面部表情和服装细节，背景可以简化处理
```

### 11.3 SD/Flux 风格权重语法

虽然本插件生成的提示词不直接包含权重语法，但您可以在生成后手动添加，或将生成的提示词传递给下游图像生成节点时添加权重。

#### 常用权重语法

| 语法格式 | 示例 | 效果说明 |
|----------|------|----------|
| `(word:1.5)` | `(masterpiece:1.5)` | 将"masterpiece"的权重提升至1.5倍 |
| `(word:0.5)` | `(background:0.5)` | 将"background"的权重降低至0.5倍 |
| `[word:1.5]` | `[beautiful eyes:1.5]` | SDXL格式的权重语法 |
| `word++` | `beautiful++` | 轻微提升权重（部分模型支持） |
| `word--` | `background--` | 轻微降低权重（部分模型支持） |

#### 权重值参考

| 权重值 | 效果 | 适用场景 |
|--------|------|----------|
| 0.3-0.5 | 显著降低 | 完全不想要的元素、需要弱化的背景 |
| 0.6-0.8 | 适度降低 | 次要元素、需要减少存在感的部分 |
| 1.0 | 默认权重 | 正常元素 |
| 1.2-1.5 | 适度提升 | 重要元素、需要强调的特征 |
| 1.6-2.0 | 显著提升 | 核心元素、必须突出的特征 |
| 2.0+ | 极端提升 | 非常关键的元素，需谨慎使用 |

#### 使用示例

将本插件生成的提示词与权重语法结合：

```
原始提示词：
一位美丽的古风仙侠少女，蓝色汉服，精致面容，温柔微笑，古风庭院，樱花飘落

添加权重后：
(masterpiece:1.5), (best quality:1.5), (ultra-detailed:1.2), 
一位极其美丽的古风仙侠少女，(蓝色汉服:1.3)，(精致面容:1.4)，(温柔微笑:1.3)，
古风庭院，(樱花飘落:1.2)，(background:0.6)
```

### 11.4 负向提示词权重

### 11.6 注意事项

1. **权重值不宜过高**：过高的权重值（如 >3.0）可能导致生成不稳定或产生奇怪的结果
2. **平衡正负权重**：正向提示词和负向提示词的权重应保持合理比例
3. **测试不同组合**：建议通过多次测试找到最适合您需求的权重组合
4. **模型差异**：不同模型对权重语法的支持程度不同，需根据实际效果调整
5. **避免冲突**：不要对相互矛盾的元素同时设置高权重（如同时强调"写实"和"卡通"）
