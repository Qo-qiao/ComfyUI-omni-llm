# ComfyUI-omni-llm ASR/TTS 模型参数说明与推荐设置

本文档详细介绍了 ComfyUI-omni-llm 插件中 ASR 模型、TTS 模型、多人会话的参数设置和推荐配置，区分了节点显示的选项和后台自动调节的参数。

## 一、ASR 模型参数说明

### 1. ASR模型加载节点参数 (llama_cpp_asr_loader)

#### 节点显示选项（可手动调节）

- **ASR模型**：选择ASR模型文件，根据需要选择合适的ASR模型，支持 .gguf, .safetensors, .pt, .pth, .bin
- **运行模式**：选择运行模式，GPU模式推荐（显存充足时），可选 GPU/CPU
- **GPU模型层数**：加载到GPU的模型层数，24GB+显存：-1，16GB显存：-1，12GB显存：-1，8GB显存：20，调节范围 -1-1000
- **语言**：识别语言，根据音频内容选择，可选 auto/zh/en/ja/ko/fr/de/es
- **任务**：任务类型，transcribe（转录）/translate（翻译成英文），可选 transcribe/translate

#### 支持的ASR模型类型

- **Whisper**：检测关键字 whisper，多语言支持好，准确率高
- **Wav2Vec2**：检测关键字 wav2vec, wav2vec2，适合特定语言优化
- **Qwen Audio**：检测关键字 qwen + audio/asr，中文支持优秀
- **FunASR**：检测关键字 fun + asr，中文语音识别
- **GLM-ASR**：检测关键字 glm + asr，支持MLX格式

#### 音频输入要求

- **支持格式**：WAV、MP3、FLAC等常见音频格式
- **推荐采样率**：16000Hz 或 22050Hz
- **声道**：单声道或立体声（会自动处理）

### 2. 在推理节点中使用ASR

在统一推理节点中：
1. 将ASR模型连接到 `asr_model` 输入端口
2. 选择推理模式为 `[基础] 音频转文本 (Audio to Text)`
3. 提供音频输入到 `audio` 端口
4. 设置合适的 `asr_language`

## 二、TTS 模型参数说明

### 1. TTS模型加载节点参数 (llama_cpp_tts_loader)

#### 节点显示选项（可手动调节）

- **TTS模型**：选择TTS模型文件，根据需要选择合适的TTS模型，支持 .gguf, .safetensors, .pt, .pth, .bin
- **GPU模型层数**：加载到GPU的模型层数，24GB+显存：-1，16GB显存：-1，12GB显存：-1，8GB显存：20，调节范围 -1-1000
- **采样率**：音频采样率（Hz），Qwen3-TTS推荐24000Hz，其他模型22050Hz，调节范围 8000-48000
- **音色**：选择音色类型，根据内容和场景选择，可选 女声1/女声2/男声1/男声2
- **情绪**：选择情绪风格，根据内容情感选择，可选 默认/开心/悲伤/愤怒/惊讶/平静/兴奋/温柔

#### 可选参数（展开后显示）

- **temperature**：生成温度，值越高越随机，默认值 0.7，调节范围 0.1-2.0
- **top_p**：核采样参数，控制生成多样性，默认值 0.9，调节范围 0.1-1.0
- **top_k**：Top-K采样参数，默认值 50，调节范围 1-100
- **repetition_penalty**：重复惩罚，值越高越避免重复，默认值 1.1，调节范围 1.0-2.0
- **max_new_tokens**：最大生成token数，默认值 2048，调节范围 100-4096
- **use_cache**：是否使用KV缓存加速推理，默认值 True，可选 True/False
- **ref_audio_path**：音色克隆参考音路径，默认值 ""（空），WAV格式文件路径
- **pitch**：音高偏移，基于参考音调整，默认值 0.0，调节范围 -5.0-5.0
- **volume**：音量，默认值 1.0，调节范围 0.1-2.0

#### 音色到speaker_id的映射

- **女声1**：speaker_id 0，适用于女性角色、温柔内容
- **女声2**：speaker_id 1，适用于女性角色、活泼内容
- **男声1**：speaker_id 2，适用于男性角色、正式内容
- **男声2**：speaker_id 3，适用于男性角色、温和内容

#### 情绪到emotion的映射

- **默认**：emotion值 default，中性情感
- **开心**：emotion值 happy，语调上扬，积极愉快
- **悲伤**：emotion值 sad，语调低沉，悲伤沉重
- **愤怒**：emotion值 angry，语调强烈，激动愤怒
- **惊讶**：emotion值 surprised，语调起伏，惊讶意外
- **平静**：emotion值 calm，语调平稳，客观冷静
- **兴奋**：emotion值 excited，语调高昂，激动兴奋
- **温柔**：emotion值 gentle，语调柔和，亲切温暖

### 2. 支持的TTS模型类型

- **Qwen3-TTS**：检测关键字 qwen + tts，高质量中文TTS，支持VoiceDesign和CustomVoice变体
- **Qwen3-TTS-MLX**：检测关键字 qwen + tts + mlx，MLX格式Qwen3-TTS，需要MLX环境
- **Bark**：检测关键字 bark，多语言支持，自然度高
- **XTTS**：检测关键字 xtts，支持音色克隆，需要参考音频
- **VITS**：检测关键字 vits，轻量级，速度快
- **Coqui**：检测关键字 coqui, your_tts，开源TTS
- **Supertonic**：检测关键字 supertonic，ONNX格式
- **Supertonic-2**：检测关键字 supertonic-2, supertonic2，改进版Supertonic

### 3. Qwen3-TTS 模型特殊说明

#### 模型变体

- **VoiceDesign**：检测关键字 voicedesign, voice_design，支持语音设计，通过自然语言指令控制音色
- **CustomVoice**：检测关键字 customvoice, custom_voice，支持音色克隆和标准音色

#### 采样率说明

- **12Hz版本**：12000Hz，路径包含"12hz"自动检测
- **标准版本**：24000Hz，默认使用

#### 音色克隆参考音要求

- **格式**：WAV格式
- **时长**：3~60秒最佳
- **声道**：单声道
- **采样率**：48000Hz
- **质量**：清晰、无噪音、无背景音乐

### 4. 在推理节点中使用TTS

#### 方式一：TTS音色美化输出模式

1. 在推理节点中选择 `音频输出模式` 为 `TTS音色美化输出`
2. 将TTS模型连接到 `tts_model` 输入端口
3. 选择合适的 `tts_voice`、`tts_emotion` 和 `tts_speed`
4. 推理节点的 `audio` 输出将包含生成的音频

#### 方式二：独立TTS合成

1. 使用TTS模型加载器加载TTS模型
2. 调用TTS模型的 `synthesize` 方法：
   ```python
   audio_output = tts_model.synthesize(
       text="要合成的文本",
       speaker_id=0,  # 音色ID
       speed=1.0,     # 语速
       emotion="default"  # 情感风格
   )
   ```
3. 使用音频保存节点保存音频

## 三、多人会话相关节点

### 1. 多人对话TTS节点 (multi_speaker_tts)

多人对话TTS功能允许您在ComfyUI中生成包含多个说话人的对话音频，支持不同的音色和情绪表达。

#### 功能特性

- 支持4种说话人：女声1、女声2、男声1、男声2
- 支持8种情绪：默认、开心、悲伤、愤怒、惊讶、平静、兴奋、温柔
- 支持多种对话文本格式
- 可调节语速、音高、音量
- 可设置说话人之间的停顿时间
- 自动拼接多个音频片段

#### 节点参数

**必需参数：**
- `tts_model`: 连接TTS加载器节点的输出
- `dialogue_text`: 多人对话文本
- `output_name`: 输出文件名

**可选参数：**
- `default_speaker`: 默认说话人（当对话中未指定时使用）
- `default_emotion`: 默认情绪（当对话中未指定时使用）
- `speed`: 语速（0.6-1.8，默认1.0）
- `pitch`: 音高偏移（-4.0到4.0，默认0.0）
- `volume`: 音量（0.2-2.0，默认1.0）
- `pause_duration`: 说话人之间的停顿时间（秒，默认0.5）
- `add_silence`: 是否添加静音（默认True）

#### 对话文本格式

支持三种对话文本格式：

##### 格式1：括号格式（推荐）

```
[女声1] 你好，今天天气怎么样？
[男声1] 今天天气很好，阳光明媚。
[女声1] 那我们出去走走吧？
[男声1] 好啊，我们去公园吧。
```

##### 格式2：冒号格式

```

##### 格式3：破折号格式

```

#### 情绪标记

#### 输出说明

节点输出两个值：

1. **audio_path**：生成的音频文件路径

     - `text`：文本内容
     - `duration`：片段时长
   - `pause_duration`：停顿时长
   - `add_silence`：是否添加静音

## 四、不同模型的推荐配置

### TTS模型推荐配置

#### 高性能TTS配置（16GB+显存）
- **推荐模型**：Qwen3-TTS、Bark
- **配置参数**：
  ```
  device_mode: GPU
  n_gpu_layers: -1
  sample_rate: 24000
  temperature: 0.7
  top_p: 0.9
  top_k: 50
  ```

#### 平衡TTS配置（8-16GB显存）
- **推荐模型**：ChatTTS、XTTS
- **配置参数**：
  ```
  device_mode: GPU
  n_gpu_layers: 20
  sample_rate: 22050
  temperature: 0.7
  ```

#### 轻量级TTS配置（<8GB显存）
- **推荐模型**：VITS、Supertonic
- **配置参数**：
  ```
  device_mode: CPU
  sample_rate: 22050
  ```

### ASR模型推荐配置

#### 高精度ASR配置
- **推荐模型**：Whisper Large-v3、Qwen-Audio
- **配置参数**：
  ```
  device_mode: GPU
  n_gpu_layers: -1
  language: auto
  task: transcribe
  ```

#### 快速ASR配置
- **推荐模型**：Whisper Base/Small、FunASR
- **配置参数**：
  ```
  device_mode: GPU
  n_gpu_layers: 20
  language: zh  # 指定语言可提高速度
  task: transcribe
  ```

## 五、性能优化建议

### 1. GPU模式优化
- 设置合适的`n_gpu_layers`，确保模型完全加载到GPU
- Flash Attention 已根据GPU类型自动启用（NVIDIA启用，AMD禁用）
- 批处理参数（n_batch, n_ubatch, n_threads, n_threads_batch）已根据硬件性能自动优化

### 2. CPU模式优化
- 线程数已根据CPU核心数自动设置
- 内存映射已默认启用以减少内存占用
- 选择轻量级模型提高推理速度

### 3. 显存管理
- 监控显存使用情况，避免OOM错误
- 对于大型模型，考虑使用CPU模式

### 4. 音频处理优化
- **采样率选择**：
  - 8000Hz：电话音质，文件最小
  - 16000Hz：标准音质，适合一般应用
  - 22050Hz：平衡音质和文件大小（推荐）
  - 44100Hz：CD音质，适合高品质音频
  - 48000Hz：专业音质，TTS参考音频推荐

- **TTS语速调节**：
  - 0.5-0.7：极慢速，适合听力障碍用户
  - 0.8-0.9：慢速，适合学习材料
  - 1.0：正常语速，适合大多数场景
  - 1.1-1.3：快速，适合新闻播报
  - 1.4-2.0：极快速，适合特殊效果

### 5. 音频处理任务优化
- **ASR+TTS组合**：可用于语音翻译、语音转换等任务
- **多人会话**：使用多人对话TTS节点实现多角色对话合成


## 六、常见问题与解决方案

### 1. TTS音频输出失败

**原因**：
- TTS模型不支持指定的音色或情绪
- 文本长度超过模型限制
- 缺少必要的音频处理依赖

**解决方案**：
- 检查TTS模型支持的音色和情绪范围
- 缩短输入文本长度
- 安装必要的音频处理库

### 2. TTS音色克隆效果不佳

**原因**：
- 参考音频质量差
- 参考音频格式不正确
- 参考音频时长不合适

**解决方案**：
- 使用清晰的参考音频，避免噪音和背景音乐
- 确保参考音频为WAV格式，单声道，48000Hz
- 参考音频时长控制在3~60秒

### 3. ASR识别准确率低

**原因**：
- 音频质量差
- 语言设置不匹配
- 模型选择不当

**解决方案**：
- 使用清晰的音频文件
- 设置正确的语言参数
- 选择适合语言和场景的ASR模型

### 4. 音频保存失败

**原因**：
- 音频数据格式不正确
- 输出路径不存在
- 格式不支持

**解决方案**：
- 确保音频数据包含waveform和sample_rate字段
- 检查输出路径是否存在
- 使用支持的格式（wav/mp3/flac/ogg）



## 七、音频时间对齐相关节点

### 1. 强制对齐模型加载器 (llama_cpp_forced_aligner_loader)

强制对齐模型加载节点用于加载Qwen3-ForcedAligner模型，支持自动检测和下载缺失文件。

#### 节点参数

**必需参数：**
- `forced_aligner_model`: 选择强制对齐模型文件，支持Qwen3-ForcedAligner-0.6B模型
- `n_gpu_layers`: 加载到GPU的模型层数，-1表示全部加载到GPU

**可选参数：**
- `precision`: 模型精度选择，可选 float16/bfloat16/float32，默认 float16
- `batch_size`: 批处理大小，默认32，范围1-128
- `flash_attention`: 是否启用FlashAttention2优化，需要安装flash-attn

#### 支持的模型
- **Qwen3-ForcedAligner-0.6B**: 阿里通义千问强制对齐模型，支持中文语音与文本的精确时间对齐

#### 模型文件要求
- `model.safetensors`: 模型权重文件
- `config.json`: 模型配置文件
- `tokenizer.json`: 分词器配置文件
- `preprocessor_config.json`: 预处理配置文件

#### 自动下载功能
节点会自动检查必备文件，缺失时会根据网络连接情况从ModelScope或Hugging Face自动下载。

### 2. 强制对齐推理节点 (llama_cpp_forced_aligner_inference)

强制对齐推理节点将音频和文本输入ForcedAligner，输出细粒度时间戳（音素/单词级）。

#### 节点参数

**必需参数：**
- `aligner_model`: 连接强制对齐模型加载器的输出
- `audio`: 输入音频数据
- `text`: 要对齐的文本内容

**可选参数：**
- `sample_rate`: 音频采样率，默认16000Hz，范围8000-48000Hz
- `output_format`: 输出格式，可选 Text/SRT/VTT/JSON，默认Text

#### 输出说明

节点输出三个值：
1. **aligned_text**: 对齐后的文本
2. **timestamps**: 时间戳数据（JSON格式），包含段落、单词和音素级别的时间信息
3. **subtitle_text**: 字幕格式文本（SRT或VTT格式）

#### 时间戳数据结构
```json
{
"task_id": "任务ID",
"audio_info": {
    "audio_duration": 音频时长(秒),
    "audio_sample_rate": 采样率
},
"segments": [
    {
    "start": 开始时间(秒),
    "end": 结束时间(秒),
    "text": "段落文本",
    "words": [
        {
        "word": "单词",
        "start": 开始时间(秒),
        "end": 结束时间(秒),
        "phonemes": [
            {
            "phoneme": "音素",
            "start": 开始时间(秒),
            "end": 结束时间(秒)
            }
        ]
        }
    ]
    }
],
"status": "success/failed",
"error_msg": "错误信息"
}
```

### 3. TTS时间对齐节点 (tts_align)

TTS时间对齐节点使用强制对齐推理节点输出的时间戳数据，实现按文本时间轴生成配音，支持精确时间对齐和自然流对齐两种模式。

#### 节点参数

**必需参数：**
- `tts_model`: TTS模型
- `timestamps`: 时间戳数据，可从强制对齐推理节点获取或导入JSON格式文件
- `text`: 要合成的文本内容
- `sample_rate`: 音频采样率，默认24000Hz，范围8000-48000Hz

**可选参数：**
- `speed`: TTS速度，默认1.0，范围0.5-2.0
- `align_mode`: 对齐模式，可选 Exact Timing(精确时间对齐)/Natural Flow(自然流对齐)，默认Natural Flow
- `silence_padding`: 段落后静音填充（秒），默认0.1，范围0.0-1.0

#### 对齐模式说明
- **Exact Timing**: 严格按照时间戳生成音频，确保每个段落的时长与时间戳完全匹配
- **Natural Flow**: 保持自然语音节奏，不强制调整音频时长

#### 输出说明
节点输出两个值：
1. **aligned_audio**: 对齐后的音频
2. **alignment_info**: 对齐信息，包含每个段落的对齐状态和时长信息



### 5. 角色配置节点 (RoleConfig)

角色配置节点用于配置角色的参数，包括名称、音色、语速、音高、音量等，支持本地TTS模型。

#### 节点参数

**必需参数：**
- `role_name`: 角色名称

**本地模型参数：**
- `voice`: 本地模型音色，可选：
  - Vivian - 明亮略带锐气的年轻女声 (中文)
  - Serena - 温暖柔和的年轻女声 (中文)
  - Uncle_Fu - 音色低沉醇厚的成熟男声 (中文)
  - Dylan - 清晰自然的北京青年男声 (中文北京方言)
- `emotion`: 本地模型情绪，可选 default/happy/sad/angry/surprised/calm/excited/gentle，默认default

**通用参数：**
- `speed`: 语速，默认1.0，范围0.5-2.0
- `pitch`: 音高偏移，默认0.0，范围-5.0-5.0
- `volume`: 音量，默认1.0，范围0.1-2.0

#### 输出说明
节点输出一个值：
- **role_config**: 角色配置字典，包含所有配置参数

## 八、使用流程示例

### 示例1：TTS语音合成

1. **加载模型**：
   - 使用 `llama_cpp_model_loader` 节点加载LLM模型
   - 使用 `llama_cpp_tts_loader` 节点加载TTS模型（如Qwen3-TTS）

2. **配置统一推理节点**：
   - 使用 `llama_cpp_unified_inference` 节点
   - 选择推理模式：`[基础] 文本转音频 (Text to Audio)`
   - 连接LLM模型到 `Llama模型` 输入
   - 连接TTS模型到 `tts_model` 输入
   - 设置TTS音色（tts_voice）、情感（tts_emotion）和语速（tts_speed）

3. **提供文本输入**：
   - 在 `文本输入` 中输入要合成的文本
   - 选择合适的预设模板

4. **保存音频**：
   - 使用 `llama_cpp_audio_saver` 音频保存节点
   - 连接推理节点的 `audio` 输出到音频保存节点
   - 选择保存格式（推荐WAV）

### 示例2：ASR语音识别

1. **加载模型**：
   - 使用 `llama_cpp_model_loader` 节点加载LLM模型
   - 使用 `llama_cpp_asr_loader` 节点加载ASR模型（如Qwen3-ASR）

2. **配置统一推理节点**：
   - 使用 `llama_cpp_unified_inference` 节点
   - 选择推理模式：`[基础] 音频转文本 (Audio to Text)`
   - 连接LLM模型到 `Llama模型` 输入
   - 连接ASR模型到 `asr_model` 输入
   - 设置ASR语言（asr_language）

3. **提供音频输入**：
   - 连接音频输入到 `audio` 端口

4. **运行识别**：
   - 获取识别文本输出

### 示例3：多人会话合成

1. **加载TTS模型**：
   - 使用 `llama_cpp_tts_loader` 节点加载TTS模型（如Qwen3-TTS）

2. **配置多人对话TTS节点**：
   - 使用 `multi_speaker_tts` 节点
   - 连接TTS模型到 `tts_model` 输入
   - 在 `dialogue_text` 中输入多人对话文本
   - 设置 `output_name` 输出文件名

3. **设置对话参数**：
   - 设置 `default_speaker` 默认说话人
   - 设置 `default_emotion` 默认情绪
   - 调整 `speed` 语速参数
   - 设置 `pause_duration` 说话人之间的停顿时间

4. **生成对话音频**：
   - 运行工作流
   - 音频文件将保存在ComfyUI输出目录的audio子目录



## 九、总结

本文档提供了 ComfyUI-omni-llm 插件中 ASR 模型、TTS 模型、多人会话的详细参数说明和推荐设置，区分了节点显示的选项和后台自动调节的参数。

### 关键要点

1. **ASR模型**：
   - 支持多种ASR架构（Whisper、Qwen-Audio等）
   - 注意选择合适的语言和任务类型
   - 音频质量直接影响识别准确率

2. **TTS模型**：
   - 支持多种TTS架构（Qwen3-TTS、Bark、XTTS等）
   - 注意采样率设置和参考音频格式
   - 音色克隆需要高质量的参考音频

3. **多人会话**：
   - 支持多角色对话合成
   - 支持不同音色和情绪表达
   - 支持多种对话文本格式
   - 可调节语速、音高、音量和停顿时间



### 使用建议

1. 从默认配置开始，根据实际效果逐步调整
2. 优先考虑硬件性能，选择适合的模型和参数
3. 注意监控资源使用情况，避免资源耗尽
4. 根据任务类型选择合适的模型和参数组合
5. 对于复杂任务，考虑使用更高性能的硬件或更适合的模型
